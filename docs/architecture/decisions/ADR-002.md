# ADR-002: In-Memory Graph with SQLite Persistence

## Context

Need fast graph traversal for code relationships. PostgreSQL adds deployment complexity.

## Decision

Use ConcurrentDictionary-based in-memory graph with SQLite for persistence. No PostgreSQL dependency.

## Alternatives Considered

1. PostgreSQL graph - Rejected: deployment complexity
2. Neo4j - Rejected: additional dependency
3. File-based graph - Rejected: slow traversal

## Consequences

- ✅ Fast random access for BFS/DFS
- ✅ No external database dependency
- ✅ Works offline
- ❌ Limited by available RAM for very large graphs
- ❌ No concurrent write support
