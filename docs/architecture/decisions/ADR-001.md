# ADR-001: Use Roslyn Instead of MSBuildWorkspace

## Context

Need to analyze C# source code semantically. MSBuildWorkspace requires MSBuild runtime which has compatibility issues with .NET 10.

## Decision

Use `CSharpCompilation` directly instead of `MSBuildWorkspace`. Parse .csproj XML manually to get source files and references.

## Alternatives Considered

1. MSBuildWorkspace - Rejected due to .NET 10 compatibility issues
2. Third-party parsers - Rejected to minimize dependencies

## Consequences

- ✅ No MSBuild runtime dependency
- ✅ Faster startup
- ✅ Works with .NET 10
- ❌ Manual .csproj parsing
- ❌ No NuGet package resolution (simplified)
